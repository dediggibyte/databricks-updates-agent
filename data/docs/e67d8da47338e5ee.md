Beta

This feature is in [Beta](/aws/en/release-notes/release-types). Workspace admins can control access to this feature from the **Previews** page. See [Manage Databricks previews](/aws/en/admin/workspace-settings/manage-previews).

The managed Outlook connector in Lakeflow Connect allows you to ingest email data from Microsoft Outlook into Databricks.

## Feature availability

| Feature | Availability |
| --- | --- |
| UI-based pipeline authoring | Supported |
| API-based pipeline authoring | Supported |
| Declarative Automation Bundles | Supported |
| Incremental ingestion | Supported |
| Unity Catalog governance | Supported |
| Orchestration using Databricks Workflows | Supported |
| SCD type 2 | Not supported |
| Column selection and deselection | Supported |
| API-based row filtering | Not supported |
| Automated schema evolution: New and deleted columns | Not supported |
| Automated schema evolution: Data type changes | Not supported |
| Automated schema evolution: Column renames | Not supported |
| Automated schema evolution: New tables | N/A |
| Maximum number of tables per pipeline | 1 |

## Start ingesting from Microsoft Outlook

The following table provides an overview of the end-to-end Outlook ingestion flow, based on user type:

| User | Steps |
| --- | --- |
| Admin | 1. Configure Microsoft Outlook to enable authentication from Databricks. See [Configure authentication to Microsoft Outlook](/aws/en/ingestion/lakeflow-connect/outlook-source-setup). 2. Either:    * Use Catalog Explorer to create a connection to Microsoft Outlook so that non-admins can create pipelines. See [Create an Outlook connection](/aws/en/ingestion/lakeflow-connect/outlook-connection).    * Use the data ingestion UI to create a connection and a pipeline at the same time. See [Ingest data from Microsoft Outlook](/aws/en/ingestion/lakeflow-connect/outlook-pipeline). |
| Non-admin | Use any supported interface to create a pipeline from an existing connection. See [Ingest data from Microsoft Outlook](/aws/en/ingestion/lakeflow-connect/outlook-pipeline). |
