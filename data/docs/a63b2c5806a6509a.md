**Applies to:**  Databricks SQL  Databricks Runtime 17.1 and above

Represents a geometric object (point, linestring, polygon, etc.) whose coordinate reference system is Euclidean for the first (X) and second (Y) coordinates, as well as their Z coordinate if it exists. The underlying coordinate reference system is described by the SRID value of the `GEOMETRY` type. If the underlying coordinate reference system is unknown the value `0` is used. The units of the X, Y, optionally Z coordinates are those implied by the SRID (except for the SRID value `0` in which case in depends on the user's usage context).

note

`GEOMETRY` columns are not supported as dimensions in metric views or Lakeflow Spark Declarative Pipelines materialized views because those contexts require `GROUP BY` support, which geospatial data does not currently provide. To use geospatial data in a metric view, add the column as a measure instead of a dimension.

## Syntax

```
GEOMETRY ( { srid | ANY } )
```

## Limits

A column of type `GEOMETRY(ANY)` can hold geometries whose SRID values can be different per row.
A column of type `GEOMETRY(ANY)` cannot be persisted.

About 11000 SRID values are supported. Learn more about [SRID](https://en.wikipedia.org/wiki/Spatial_reference_system#Identifier).
The user will get a [ST\_INVALID\_SRID\_VALUE](/aws/en/error-messages/error-classes#st_invalid_srid_value) error if they try to define a `GEOMETRY` column with an unsupported SRID value.

## Notes

* To convert a `GEOMETRY` value to one of the geospatial standard or widely used formats you can use:
  + [`st_asbinary` function](/aws/en/sql/language-manual/functions/st_asbinary) to export a `GEOMETRY` in WKB format.
  + [`st_asewkb` function](/aws/en/sql/language-manual/functions/st_asewkb) to export a `GEOMETRY` in Extended WKB (EWKB) format.
  + [`st_asewkt` function](/aws/en/sql/language-manual/functions/st_asewkt) to export a `GEOMETRY` in Extended WKT (EWKT) format.
  + [`st_asgeojson` function](/aws/en/sql/language-manual/functions/st_asgeojson) to export a `GEOMETRY` in GeoJSON format.
  + [`st_astext` function](/aws/en/sql/language-manual/functions/st_astext) to export a `GEOMETRY` in WKT format.

## Examples

SQL

```
> SELECT hex(st_asbinary(st_geomfromtext('POINT(1 2)')));  
  0101000000000000000000f03f0000000000000040  
  
> SELECT st_asewkt(st_geomfromwkb(X'0101000000000000000000f03f0000000000000040'));  
  POINT(1 2)  
  
> SELECT st_asgeojson(st_geomfromtext('POINT(1 2)'));  
  {"type":"Point","coordinates":[[1,2]]}  
  
> SELECT st_astext(st_geomfromgeojson('{"type":"Point","coordinates":[[1,2]]}'));  
  POINT(1 2)  
  
> SELECT st_astext(to_geometry('{"type":"Point","coordinates":[[1,2]]}'));  
  POINT(1 2)  
  
> SELECT try_to_geometry('not a geometry value');  
  NULL
```
