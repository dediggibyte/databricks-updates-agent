This feature is in [Beta](/aws/en/release-notes/release-types). Workspace admins can control access to this feature from the **Previews** page. See [Manage Databricks previews](/aws/en/admin/workspace-settings/manage-previews).

The managed Workday Human Capital Management (HCM) connector in Lakeflow Connect allows you to ingest data from Workday HCM into Databricks. Use this connector to consolidate human resources (HR) data such as workers, positions, payroll, and talent records into Databricks for analysis and downstream processing.

This page covers the Workday HCM connector, which ingests data from standard Workday HCM modules. To ingest data from Workday custom reports, see [Workday Reports connector](/aws/en/ingestion/lakeflow-connect/workday-reports).

## Start ingesting from Workday HCM

The following table gives an overview of the end-to-end Workday HCM ingestion flow based on user persona:

| User | Steps |
| --- | --- |
| Admin | 1. Configure Workday HCM to enable authentication from Databricks. See [Configure authentication to Workday HCM](/aws/en/ingestion/lakeflow-connect/workday-hcm-setup). 2. Either:    * Use Catalog Explorer to create a connection to Workday HCM so that non-admins can create pipelines. See [Create a Workday HCM connection](/aws/en/ingestion/lakeflow-connect/workday-hcm-connection).    * Use the data ingestion UI to create a connection and a pipeline at the same time. See [Ingest data from Workday HCM](/aws/en/ingestion/lakeflow-connect/workday-hcm-pipeline). |
| Non-admin | Use any supported interface to create a pipeline from an existing connection. See [Ingest data from Workday HCM](/aws/en/ingestion/lakeflow-connect/workday-hcm-pipeline). |
