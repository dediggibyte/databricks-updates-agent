Beta

This feature is in [Beta](/aws/en/release-notes/release-types). Workspace admins can control access to this feature from the **Previews** page. See [Manage Databricks previews](/aws/en/admin/workspace-settings/manage-previews).

The managed Strac connector in Lakeflow Connect allows you to ingest data-security audit events from Strac into Databricks.

## Ingest from Strac in 3 steps

note

Before starting, review the [Databricks user persona](/aws/en/connect/managed-ingestion), [supported interfaces](/aws/en/ingestion/lakeflow-connect/faq#interfaces), [ingestion frequency](/aws/en/ldp/pipeline-mode), and [common patterns](/aws/en/ingestion/lakeflow-connect/common-patterns).

1. **[Configure Strac for ingestion](/aws/en/ingestion/lakeflow-connect/strac-source-setup)** (Admins) — Obtain a Strac API key to authenticate with Databricks.
2. **[Create a Unity Catalog connection](/aws/en/ingestion/lakeflow-connect/strac-connection)** (Admins) — Create a connection in Catalog Explorer so non-admins can create pipelines.
3. **[Create an ingestion pipeline](/aws/en/ingestion/lakeflow-connect/strac-pipeline)** (Admins or non-admins) — Select any supported interface and create a pipeline from an existing connection.
