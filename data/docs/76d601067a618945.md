Lakebase Postgres provides data protection features to help secure your databases and control access to critical data. These features enable you to protect production environments and establish secure network connectivity between your applications and databases.

## Data protection features

| Feature | Description |
| --- | --- |
| [Customer-managed keys](/aws/en/oltp/projects/customer-managed-keys) | Encrypt Lakebase project data at rest using a key that your organization owns and manages in your cloud KMS. |
| [Protected branches](/aws/en/oltp/projects/protected-branches) | Configure branch protection rules to prevent accidental modifications or deletions of critical branches, ensuring the integrity of your production and staging data. |
| [Private Link for Lakebase Autoscaling](/aws/en/oltp/projects/private-link) | If your workspace uses Private Link, Lakebase Autoscaling requires inbound Private Link. Postgres clients connecting from outside the workspace also require inbound Private Link for performance-intensive services. |

## Compliance

Lakebase Autoscaling supports workspaces with the compliance security profile when the compliance standard is set to HIPAA, C5, TISAX, or None. Other compliance standards are not supported.
