Public Preview

AI Runtime for single-node tasks is in [Public Preview](/aws/en/release-notes/release-types). The distributed training API for multi-GPU workloads remain in [Beta](/aws/en/release-notes/release-types).

AI Runtime is a compute offering at Databricks intended for [deep learning](https://www.databricks.com/blog/what-is-deep-learning) workloads, and brings GPU support for Databricks Serverless. You can use AI Runtime to train and fine-tune custom models using your favorite frameworks and get state-of-the-art efficiency, performance, and quality. For an overview of how [serverless compute](/aws/en/getting-started/high-level-architecture#serverless-compute-plane) fits into the Databricks architecture, see [Serverless workspace architecture](/aws/en/getting-started/high-level-architecture#serverless-workspace-architecture).

## Key features

* **Fully managed GPU infrastructure**: Serverless, flexible access to GPUs and no cluster configuration, driver selection, or autoscaling policies to manage.
* **A runtime dedicated for deep learning**: Choose either a minimal Standard environment for maximum flexibility over dependencies or a full-featured AI environment pre-loaded with popular ML frameworks.
* **Natively integrated** across notebooks, jobs, Unity Catalog, and MLflow for seamless development, data access, and experiment tracking.

## Hardware options

All AI Runtime accelerators provision a single node. The number of GPUs on that node depends on the accelerator type:

| Accelerator | GPUs per node | GPU memory | Best for | Distributed training |
| --- | --- | --- | --- | --- |
| 1xA10 | 1 | 24 GB | Small to medium ML and deep learning tasks such as classic ML models or fine-tuning smaller language models | Not supported (single GPU) |
| 1xH100 | 1 | 80 GB | Workloads that need more GPU memory than 1xA10 provides but do not require multi-GPU distributed training, such as fine-tuning medium-size language models | Not supported (single GPU) |
| 8xH100 | 8 | 80 GB per GPU | Large-scale AI workloads including training or fine-tuning massive models or running advanced deep learning tasks | Supported using the `@distributed` decorator with `gpus=8` |

## Recommended use cases

Databricks recommends AI Runtime for any custom model training use cases that involve deep learning, large-scale classic workloads, or GPUs.

For example:

* LLM fine-tuning (LoRA, QLoRA, full fine-tuning)
* Computer vision (object detection, image classification)
* Deep-learning-based recommender systems
* Reinforcement learning
* Deep-learning-based time series forecasting

## Requirements

* A workspace in one of the following AWS-supported regions:
  + `us-west-2`
  + `us-west-1`
  + `us-east-1`
  + `us-east-2`
  + `ca-central-1`
  + `sa-east-1`
* The AI Runtime preview must be enabled via workspace admin settings. See [Manage Databricks previews](/aws/en/admin/workspace-settings/manage-previews).

## Limitations

* AI Runtime only supports A10 and H100 accelerators.
* AI Runtime doesn't support the [compliance security profile](/aws/en/security/privacy/security-profile) for the FedRAMP High or DoD IL5 standards. All other compliance standards are supported.
* Adding dependencies using the **Environments** panel is not supported for AI Runtime scheduled jobs. Install dependencies programmatically using `%pip install` in your notebook instead.
* For scheduled jobs on AI Runtime, auto recovery behavior for incompatible package versions that are associated with your notebook is not supported.
* The maximum runtime for a workload is seven days. For model training jobs that exceed this limit, implement checkpointing and restart the job once the maximum runtime is reached.
* AI Runtime provides on-demand access to GPU resources. While this leads to easy, flexible access to GPUs, there may be periods where capacity is constrained or unavailable in your region.
* AI Runtime leverages cross-region GPUs in certain cases during moments of high demand. There may be egress costs associated with such usage, and cross-region network connectivity might be limited at certain times.

## Distributed training

Beta

This feature is in [Beta](/aws/en/release-notes/release-types). Workspace admins can control access to this feature from the **Previews** page. See [Manage Databricks previews](/aws/en/admin/workspace-settings/manage-previews).

AI Runtime supports distributed training across multiple GPUs on the single node your notebook is connected to. Using the `@distributed` decorator from the `serverless_gpu` Python API (Beta), you can launch multi-GPU workloads with PyTorch DDP, FSDP, or DeepSpeed with minimal configuration. For details, see [Multi-GPU workload](/aws/en/machine-learning/ai-runtime/distributed-training).
