## Arguments

`content` is the only required argument. All options passed in the `map` argument (`version`, `imageOutputPath`, `descriptionElementTypes`, and `pageRange`) are optional.

* `content`: A `BINARY` expression representing the input byte array data.
* `version`: The version of the output schema, supported: "2.0".
* `'imageOutputPath'`: Save rendered page images to a Unity Catalog volume for reference or multi-modal RAG applications.
* `'descriptionElementTypes'`: AI-generated descriptions. Only descriptions for `figures` are supported for version 2.0, so `'*'` and `'figure'` produce the same behavior.
  + '' (empty string): No descriptions are generated. This reduces the compute needed and costs for documents with lots of figures.
  + `'figure'`: Generate descriptions for figures only. Only supports AI-generated descriptions.
  + `'*'` (default): Generate descriptions for all supported element types.
* `'pageRange'`: Restricts parsing to a subset of pages in the document. Page numbers are 1-indexed. The value is a comma-separated list of tokens, where each token is either a single page (for example, `"3"`) or an inclusive range (for example, `"5-10"`). For example, `'1,3,5-10'` parses pages 1, 3, and 5 through 10. The selected pages must stay within the 500-page limit. If `pageRange` is omitted and the document exceeds 500 pages, the function fails immediately without parsing any pages.

## Returns

The `ai_parse_document` function extracts the contextual layout metadata from the document, like `page_number`, `header`, `footer`. It also extracts the content of the document, such as text paragraphs. For version 2.0, tables are represented in HTML. The output is of `VARIANT` type.

### Understanding elements

An **element** is a discrete unit of content identified within a parsed document. When `ai_parse_document` processes a document, it breaks the document into a sequence of elements, where each element represents a distinct content block such as a text paragraph, a table, a figure, or a layout marker like a page header or footer.

Each element in the output `elements` array includes the following fields:

* `id`: A 0-based index indicating the element's position within the document.
* `type`: A string indicating the kind of content the element represents. Supported element types are:
  + `text`: A text paragraph or general body text.
  + `table`: A table, with content represented in HTML format.
  + `figure`: An image or diagram within the document.
  + `title`: The document title.
  + `caption`: A caption associated with a figure or table.
  + `section_header`: A heading or subheading that denotes the start of a section.
  + `page_header`: A header that appears at the top of a page.
  + `page_footer`: A footer that appears at the bottom of a page.
  + `page_number`: A page number marker.
  + `footnote`: A footnote reference or text.
* `content`: The extracted text content of the element. For `table` elements, the content is formatted as HTML. For `figure` elements, the content may be `NULL`.
* `confidence`: A confidence score indicating how reliably the element was extracted from the document.
* `bbox`: An array of bounding box coordinates indicating the physical location of the element on the page. Each bounding box includes pixel coordinates and a `page_id` reference.
* `description`: An AI-generated text description. In version 2.0, descriptions are only generated for `figure` elements when the `descriptionElementTypes` option is enabled.

## Migrate workloads to updated schema

The steps in this section describe how to migrate workloads that were created before September 22, 2025 to use the updated output schema.

1. In your SQL request, specify a specific schema version using the `version` parameter.

SQL

```
SELECT  
ai_parse_document(  
  content,  
  map('version', '2.0')  
) AS parsed  
FROM READ_FILES('/path/to/documents', format => 'binaryFile');
```

1. Modify your code to read content from the `elements` array instead of the `pages` array.
2. Re-evaluate metadata. For example, if you were using `page` metadata like headers and footers, you need to develop an alternative approach for extracting this information from the `elements`.
3. Validate your updated logic with sample documents before migrating your full workload.
4. Consider enabling figure descriptions or image persistence if they are relevant to your use case.
5. Check permissions. For example, if you plan to use image persistence, ensure you have the correct permissions set up for the target Unity Catalog volume.

The following example uses `ai_parse_document` to extract text elements and concatenate all text content. From there it uses `ai_query` with the Claude Sonnet 4 model to extract specific structured information like vendor name, date, invoice number, and items purchased.

SQL

```
WITH parsed_docs AS (  
  SELECT  
    path,  
    ai_parse_document(  
      content,  
      MAP('version', '2.0')  
    ) AS parsed_content  
  FROM READ_FILES('/Volumes/finance/invoices/', format => 'binaryFile')  
)  
SELECT  
  path,  
  ai_extract(  
    parsed_content,  
    '["invoice_id", "vendor_name", "total_amount"]',  
    MAP('instructions', 'These are vendor invoices.')  
  ) AS invoice_data  
FROM parsed_docs;
```

### Using to\_json() with PySpark collect()

`ai_parse_document` returns a `VARIANT` type, which cannot be directly collected by PySpark (or other APIs that do not support VARIANT). To collect parsed results into Python for further processing, use `to_json()` in SQL to convert the VARIANT to a JSON string, then parse it with `json.loads()` in Python:

Python

```
import json  
  
sql = """  
WITH parsed_documents AS (  
  SELECT  
    path,  
    ai_parse_document(  
      content,  
      map(  
        'version', '2.0',  
        'imageOutputPath', '/Volumes/catalog/schema/volume/parsed_images/',  
        'descriptionElementTypes', '*'  
      )  
    ) AS parsed  
  FROM READ_FILES('/Volumes/catalog/schema/volume/source_docs/*', format => 'binaryFile')  
)  
SELECT path, to_json(parsed) AS parsed_json FROM parsed_documents  
"""  
parsed_results = [json.loads(row.parsed_json) for row in spark.sql(sql).collect()]  
# Each item in parsed_results is a Python dict with the parsed document structure.
```
