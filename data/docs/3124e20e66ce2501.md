This page introduces AI assistive features in Databricks that can help you work more efficiently.

## What are Databricks AI assistive features?

Databricks AI assistive features are ready-to-use tools available in Databricks workspaces that provide intelligent support and assistance with various tasks.

Databricks AI assistive features comprise of:

| AI assistive feature | Description |
| --- | --- |
| [Genie Code](/aws/en/genie-code/) | An AI assistant that helps users interact with their data, notebooks, and workflows by providing intelligent, context-aware responses and automation. |
| [Genie Agents](/aws/en/genie/) | Set up Genie Agents to explore chosen datasets through natural language. Ask Genie questions about your company’s data to analyze information and gain business insights. |
| [Genie Ontology](/aws/en/genie-one/chat#ontology) | A map of your data and business that Genie automatically generates and maintains to return more accurate, lower-latency responses. |
| [AI-generated comments](/aws/en/comments/ai-comments) | Auto-generated comments that describe your Unity Catalog objects, including tables, columns, catalogs, and schemas using metadata. |
| [Intelligent search](/aws/en/search/#intelligent-search) | AI-powered search experience that uses Unity Catalog comments to provide more context-aware search results. |

These features use compound AI systems that combine the use of AI models, retrieval, ranking, and personalization systems to understand your organization's data and usage patterns. They enable everyone in an organization to be more productive using data and AI, while maintaining the governance and controls established in Unity Catalog.

## Partner-powered AI features

By default, Databricks AI assistive features are partner-powered by Azure OpenAI service, OpenAI on Databricks, and Anthropic on Databricks. Admins can choose to disable the **partner-powered AI features** setting. When the setting is disabled, Databricks AI assistive features use Databricks-hosted models where available, or are disabled entirely. For more information, see [Partner-powered AI features](/aws/en/databricks-ai/partner-powered).

## Databricks-hosted models for AI-powered features

Databricks recommends using [Partner-powered AI features](/aws/en/databricks-ai/partner-powered) for the best experience with AI assistive features. However, workspaces admins can set up their workspaces to use Genie Code fully powered by models hosted and served directly on Databricks infrastructure, at no additional cost.

Databricks-hosted models for Genie Code is the default experience for workspaces in the Americas and Europe that have disabled [Partner-powered AI features](/aws/en/databricks-ai/partner-powered). For workspaces in other geographies to use Databricks-hosted models for AI-powered features, the workspace must have both **[Enforce data processing within workspace Geography for Designated Services](/aws/en/resources/databricks-geos#cross-geo-processing)** and **partner-powered AI features** settings **disabled**.

See [Databricks AI assistive features trust and safety](/aws/en/databricks-ai/databricks-ai-trust).
