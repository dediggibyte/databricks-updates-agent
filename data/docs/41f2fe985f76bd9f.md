Ask natural-language questions of your company data in a Genie Agent, a no-code, self-service chat interface for business users. Read the SQL-backed responses, view auto-generated visualizations, and leave feedback that agent authors use to refine the agent over time. This page covers the end-user experience; for agent setup, see [Create and manage a Genie Agent](/aws/en/genie/set-up).

note

Genie Agents were formerly known as Genie Spaces.

## Overview

Genie provides a conversational interface for querying your data using natural language. It uses generative AI tailored to your organization's business terminology and data and can incorporate user feedback to refine responses over time.

Genie offers the following key capabilities:

* Enables users to ask natural language questions to query enterprise data directly.
* Provides relevant responses derived directly from organizational data.
* Supports self-service to address questions beyond the scope of dashboards and reports, without requiring advanced tools or expertise.

The Genie Agent UI includes the following areas:

* **New chat:** Use the chat window to ask questions and read Genie's responses. It is open by default when you open a Genie Agent.
* **Chat history:** Your conversations are saved in threaded chat history. Each chat thread maintains context about previous questions.
* **Context** Shows information about the Genie Agent across three tabs:
  + **About**: Owner, tags, description, and common questions.
  + **Data**: Tables selected for the agent. Expand each table to view column names and descriptions. Click the  button to view table details in Catalog Explorer. To learn more, see [What is Catalog Explorer?](/aws/en/catalog-explorer/).
  + **Instructions**: Text instructions, joins, common SQL expressions, and SQL queries and functions.
* **Share:** Get a shareable link to a Genie Agent.
* **Monitoring** (accessed via the  kebab menu): Shows individual Genie interactions, not threaded by topic. You can rate responses or request reviews from someone with elevated permissions on the agent. Quality ratings and review correspondence appear in the history tab.

Think of Genie as a new employee who improves with clear guidance and consistent feedback. When it answers incorrectly, it's helpful to ask Genie follow-up questions and clarifications to enhance Genie's understanding within that conversation. It's also crucial to provide specific feedback that agent managers can use to refine Genie's overall knowledge and instructions, which helps develop a more reliable Genie Agent for all conversations.

## Start a new chat

Click an example question or enter a new question to start a conversation. Each conversation thread maintains its context, letting you build on previous questions. For example, if you ask, “Show me sales by product line,” you can refine results in the same thread by adding, “Only for July 2024.”

note

Context from earlier conversations in a thread is used to inform later conversations in the same thread. This context does not carry over to other chats.

Click the  New chat button to start a new conversation. By default, the thread is named with the initial prompt.

## What questions can I ask?

Genie's model for translating text into SQL uses Chain-of-Thought reasoning to break each question into clear steps. First, it identifies the relevant columns and instructions; next, it plans the SQL generation; and finally, it combines the parts into a single query.

This approach enables Genie to select precise filter conditions and address nuanced questions. However, it does not interpret query results or offer recommendations. For the best outcomes, ask specific, data-driven questions rather than seeking explanations or advice.

The table below outlines example question types Genie can answer and non-examples of questions it cannot process effectively.

| Questions Genie can answer | Questions Genie cannot answer |
| --- | --- |
| What were the sales this quarter? | Why were sales low this quarter? |
| Which customers generated the most revenue? | How can I get more revenue from Customer M? |
| Which sales representatives generated the most revenue? | What incentives should I offer my sales team to generate more revenue? |
| What were the top-performing marketing campaigns in 2024? | How can I improve my marketing strategy? |
| Which sales regions generated the most profit? | What are the best sales regions? |

For both straightforward data questions and complex, exploratory questions that investigate causes and patterns, use Agent mode. See [Agent mode in Genie Agents](/aws/en/genie/agent-mode).

## Define terms in a conversation

When you enter a question, Genie uses table and column metadata, instructions provided by agent editors, and context from the current conversation thread to generate a response. If Genie can't answer the question, it might respond by asking a follow-up clarification question. Or, it might explain that it can't answer based on the provided information. You can continue to work with Genie to get an answer to your question by defining words, relationships, or business jargon in the conversation.

For example, if you ask something like, *“How many customers churned in the past year?”* Genie might need additional information, such as how *churn* is defined. It could respond with a clarifying question or state its limitations. You can then provide more context in the same conversation, which Genie uses to improve its understanding for future interactions in that thread.

note

Genie, like other large language models (LLMs), can exhibit non-deterministic behaviors. This means that you might receive different outputs when submitting the same prompt multiple times.

## Edit your question

Edit your question to revise your original message and quickly iterate on a response. Refining your prompt helps Genie improve the specific answer without recalculating everything from scratch. Because Genie uses previous context to inform later responses, editing also helps keep the conversation focused and relevant. To edit a question, hover over the question you want to edit and click the pencil icon.

## Review a response

Genie uses the provided context and instructions to interpret your question and typically returns a brief natural language response. It may also ask clarifying questions before providing a final answer.

If the response includes a generated SQL query, the output can include the following components:

* **Analysis**: Expand this section to see how Genie interpreted your prompt. The analysis includes several components:

  + **Understanding the question**: Shows how Genie interpreted your question and the key terms it identified.
  + **Found relevant data**: When applicable, shows the specific tables or data sources Genie identified as relevant to your question.
  + **Referenced SQL example**: When applicable, displays a relevant SQL example that Genie used as a reference.
  + **Thinking steps**: Outlines the logical steps that led to the SQL query used to generate the results. These steps explain the SQL logic in plain language to help non-technical users understand and build trust in how answers are generated.

  Reviewing this section can help confirm whether the results align with your intended purpose.
* **Summary**: Genie automatically generates a natural language summary of the SQL execution results, making the response easier to interpret and more conversational.
* **Result table**: Displays the results from the generated query in table format. Click **Show code** to inspect the query Genie generated.
* **Associated visualizations**: When applicable, Genie generates a visualization of the result set. To edit the visualization, see [Edit visualizations](#edit-vis).
* **Applied filters**: Shows any filters applied to the data. You can edit these values to change the result set.

### Provide feedback

Users with CAN MANAGE permissions can monitor all user interactions within a Genie Agent. They can review questions, responses, and submitted feedback, and respond directly to users when a response is marked for review. Your feedback helps improve the agent and clarify user intent, so it's important to use the built-in tools to assess response quality.

note

Feedback is visible to users with CAN MANAGE permissions. Genie does not automatically learn or change its behavior in response to feedback.

Positive feedback on responses that join tables or use SQL expressions can prompt Genie to suggest new SQL snippets (such as measures, joins, or filters) to agent managers for review and approval. To improve future responses in other ways, request a review and include specific corrections or additional context for an agent editor.

To provide feedback on a response, choose one of the following options:

* **Yes:** Select **Yes** if the response appears accurate.
* **Fix it:** Select **Fix it** if the response is incorrect. A list of common issues appears. Choose reasons that describe why you think the response is incorrect or enter your own in the text field. Then do one of the following:
  + Click **Submit and try again** to provide feedback and regenerate the response. Genie tries to use your feedback to help generate a new response.
  + Click **Submit** to report the issue without regenerating the response.
* **Request review:** Select this option to flag the response for manual review. Optionally, add a comment to give editors additional context.

### Trusted assets

In Chat mode, when Genie answers using a trusted asset, it found a query or function that an agent editor included in the agent's instructions. This means an editor anticipated your question, or something similar, and wrote a SQL query or registered a Unity Catalog function to answer it. You can learn more about the query or function by clicking **Show more** in the response.

Trusted functions: Clicking **Show more** displays the comments provided by the function creator. These comments typically include an explanation of what the function is designed to do and sample questions it is meant to answer. If a parameter was passed to the function, the parameter value is also shown.

Trusted queries: Clicking **Show more** reveals the parameter value used to generate the response. You can enter a new value and rerun the query to adjust the response.

note

Occasionally, Genie can match your question to an incorrect function or query. Review the details to check that the provided answer corresponds to your question.

### Suggested follow-up questions

After each response, Genie uses agent instructions and conversation context to generate suggested follow-up questions. These suggestions guide you through a comprehensive line of questioning, helping you uncover key aspects of your data that you might not have thought to ask about. Click a suggested follow-up question to get a response.

## Generate visualizations and tables

For many questions, Genie automatically generates a visualization and a result table. You can ask for a specific visualization to be included in a response. To hide an automatically generated visualization, click the  collapse button. To show it again, click **Show visualization**.

### Edit visualizations

You can edit visualizations in the chat window.

1. In the upper-right corner of the visualization, click the  edit button. An editing pane opens on the right side of the page.
2. Select a different chart type from the **Visualization** drop-down menu.
3. Configure the visualization using the options in the editing pane. For details on configuration options by chart type, see [AI/BI dashboard visualization types](/aws/en/dashboards/manage/visualizations/types).

### Save a visualization to a dashboard

You can save a visualization from a Genie Agent response to a new or existing dashboard.

1. In the upper-right corner of the visualization, click the  button.
2. Select one of the following:
   * **Create new dashboard**: Enter a name for the new dashboard, then click **Create**.
   * **Add to existing dashboard**: Select an existing dashboard from the list, then click **Import**.

### Format results

To format a column in a result table, click the  kebab menu in the column heading, then click **Format** to view the available options. Formatting options depend on the column's data type.

### Download results and visualizations

To download results data as a CSV, click the  download button in the response. You can download up to approximately 1GB of data. The final file size might be slightly more or less than 1GB, as the limit is applied to an earlier step than the final file download.

To download a visualization as a PNG, click the  download button on the visualization.

### Copy results to clipboard

To copy results to your clipboard, click the  clipboard button in the response.

## Review chats

Click the  history icon to view your chat history. A record of your threaded conversations are stored and accessible until the agent is deleted. Most users can only view their own history. Each chat thread maintains its own context, meaning early messages inform subsequent questions and responses only within that thread. Context established in one chat does not carry over to others. You can open past chats to refine your questions or insights, enabling you to build more complex analyses and gain deeper understanding over time.

To edit the name of a chat:

1. Hover over the conversation thread and click the  edit icon.
2. Enter a new name into the text field. Press enter to save the new name.

To delete a chat:

1. Hover over the thread and click the  trash icon.
2. Click **Confirm and Delete Chat** to remove the chat from your history.

note

Users with elevated privileges on the agent can view all chats. Deleting a chat from your chat history also deletes it from their history page.

To review a previous response:

1. Open the conversation by clicking its name.
