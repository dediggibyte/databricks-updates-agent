As a workspace admin user, you can configure when users get emails for certain events. To manage the email settings:

1. Go to the [settings page](/aws/en/admin/admin-concepts#admin-settings).
2. Click the **Notifications** tab.
3. Enable or disable your desired email settings.

You can also update the notification destinations from this page. For more information on notification destinations, see [Manage notification destinations](/aws/en/admin/workspace-settings/notification-destinations).

## Available email settings

* **Enable dashboard email subscriptions**: Enable users to subscribe to dashboards in the workspace. See [Workspace admin subscription controls](/aws/en/ai-bi/admin/#email-subscriptions).
* **MLflow model registry email notification**: Enable or disable emails about activity on registered models and model versions for the workspace. See [Control notification preferences](/aws/en/machine-learning/manage-model-lifecycle/workspace-model-registry#email-notification).
* **Enable default email destinations for access requests in UC**: Enable default email destinations to ensure that access requests are delivered even when no destination is manually configured for an object. See [Manage access requests](/aws/en/data-governance/unity-catalog/manage-privileges/access-request-destinations).

* **Operational emails**: Specify an additional recipient for operational emails about this workspace, such as notifications about behavior or pricing changes, rollout announcements, and other events that workspace admins need to be aware of. When configured, the address receives the same operational emails as workspace admins. To send to multiple people, point this setting at a distribution-list alias.
