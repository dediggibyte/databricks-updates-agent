**Applies to:**  Databricks SQL  Databricks Runtime

Returns the basic metadata information of a table.
The metadata information includes column name, column type and column comment.
Optionally, you can specify a partition spec or column name to return the metadata pertaining to a partition or column respectively.
With Delta tables, not all fields are returned.

The metadata is either returned in report form or as a JSON document.

important

The non-JSON output of `DESCRIBE TABLE` can change, and new columns might be appended over time. Avoid relying on the output meeting a specific size or schema. For programmatic consumption, use `DESCRIBE TABLE ... AS JSON`.

## Syntax

```
{ DESC | DESCRIBE } [ TABLE ] [ EXTENDED ] table_name { [ PARTITION clause ] | [ column_name ] } [ AS JSON ]
```

For compatibility `FORMATTED` can be specified as a synonym for `EXTENDED`.

## Parameters

* **`EXTENDED`**

  If specified display detailed information about the specified columns, including the column statistics collected by
  the command, and additional metadata information (such as schema qualifier, owner, and access time).
* **[table\_name](/aws/en/sql/language-manual/sql-ref-names#table-name)**

  Identifies the table to be described. The name may not use a [temporal specification or options specification](/aws/en/sql/language-manual/sql-ref-names#table-name).
  If the table cannot be found Databricks raises a [TABLE\_OR\_VIEW\_NOT\_FOUND](/aws/en/error-messages/table-or-view-not-found-error-class) error.
* **[PARTITION clause](/aws/en/sql/language-manual/sql-ref-partition#partition)**

  An optional parameter directing Databricks SQL to return addition metadata for the named partitions.
* **[column\_name](/aws/en/sql/language-manual/sql-ref-names#column-name)**

  An optional parameter with the column name that needs to be described.
  Currently nested columns are not allowed to be specified.

  JSON format is supported for individual columns.
* **`AS JSON`**

  **Applies to:**  Databricks SQL  Databricks Runtime 16.2 and above

  Optionally returns the table metadata as a JSON string instead of a human-readable report.
  Use this format when parsing the result using a program.

  Only supported when `EXTENDED` format is specified.

Parameters `partition_spec` and `column_name` are mutually exclusive and cannot be specified together.

note

For a parameterized metric view, `DESCRIBE TABLE` lists the view parameters after the column metadata under a `# Parameters` heading, reusing the `col_name`, `data_type`, and `comment` fields. Parameter default values are not shown. With `AS JSON`, parameters are returned in a `parameters` field that includes each parameter's `name`, `type`, and `default`. Parameters require Databricks Runtime 18.2 and above.
