This page describes how to use Knowledge Assistant to create a question-and-answer chatbot over your documents and improve its quality based on natural language feedback from your subject matter experts.

## What is Knowledge Assistant?

Use Knowledge Assistant to create a chatbot that can answer questions about your documents and provide high-quality responses with citations. Knowledge Assistant uses advanced AI and follows an [Instructed Retriever](https://www.databricks.com/blog/instructed-retriever-unlocking-system-level-reasoning-search-agents) approach to address limitations of the traditional retrieval-augmented generation (RAG) approach and deliver the highest quality answers based on the documentation you provide. It supports documents in multiple languages.

Knowledge Assistant is ideal for supporting the following use cases:

* Answer user questions based on product documentation.
* Answer employee questions related to HR policies.
* Answer customer inquiries based on support knowledge bases.

Knowledge Assistant enables you to improve the chat agent's quality and adjust its behavior based on natural language feedback from your subject matter experts. Provide questions and guidelines directly in the experience, and share your agent to let others collaborate and improve the agent's performance.

Knowledge Assistant creates an agent endpoint that you can use downstream for your applications. For example, the image below shows how you can interact with the endpoint by chatting with it in AI Playground. Ask the agent questions related to your documents, and the agent will answer with citations.

Knowledge Assistant uses [default storage](/aws/en/storage/default-storage) to store temporary data transformations, model checkpoints, and internal metadata that power each agent. On agent deletion, all data associated with the agent is removed from default storage.

## Requirements

* A workspace that includes the following:
  + [Serverless compute](/aws/en/compute/serverless/) available (enabled by default in workspaces with Unity Catalog).
  + Unity Catalog enabled. See [Enable a workspace for Unity Catalog](/aws/en/data-governance/unity-catalog/enable-workspaces).
  + Access to [Model Serving](/aws/en/machine-learning/model-serving/).
  + Access to a [serverless usage policy](/aws/en/admin/usage/budget-policies) with a nonzero budget.
* A workspace in one of the [supported regions](/aws/en/resources/feature-region-support#ai-and-machine-learning-features-availability).
* You must have input data ready to use. You can choose from the following:
  + Files in a Unity Catalog volume or volume directory. Supported file types are txt, pdf, md, ppt/pptx, and doc/docx.
  + A Unity Catalog table with a file column. See [Step 1: Configure your agent](#step-1-configure-your-agent) for schema requirements and configuration details.
  + An AI Search index that uses one of the following embedding models: `databricks-gte-large-en`, `databricks-bge-large-en`, or `databricks-qwen3-embedding-0-6b`. See [Create an AI Search index](/aws/en/ai-search/create-ai-search#create-vector-search-index).

## Create a knowledge assistant agent

Go to  **Agents** in the left navigation pane of your workspace. Click **Create Agent**, and select **Knowledge Assistant**.

## Step 1: Configure your agent

Configure your agent and provide knowledge sources for it to use to answer questions.

1. In the **Name** field, enter a name for your agent.
2. In the **Description** field, describe what your agent can do.
3. In the **Knowledge source** panel, add your knowledge source. You can choose to provide files in a volume, files in a table, or an AI Search index.

   * Files in a Volume
   * Files in a Table
   * AI Search Index

   For files in a Unity Catalog volume or directory, the following file types are supported: txt, pdf, md, ppt/pptx, and doc/docx. Files larger than 50 MB are automatically skipped during ingestion and are not included in the knowledge base. PDF, DOC, DOCX, PPT, and PPTX files with more than 500 pages are also skipped. For PowerPoint files, each slide counts as a page. TXT and MD files have no page limit.

   1. Under **Type**, select **Files in a Volume**.
   2. In the **Source** field, select the Unity Catalog volume or volume directory that contains your files.
   3. In the **Name** field, enter a name for your knowledge source.
   4. Under **Describe the content**, describe what content the knowledge source contains to help the agent understand when to use this data source.

   You can use files in a Unity Catalog table as a knowledge source. Tables ingested from [SharePoint](/aws/en/ingestion/sharepoint) or [Google Drive](/aws/en/ingestion/google-drive) using Databricks connectors produce the expected format automatically. For tables ingested from other sources such as [Jira](/aws/en/ingestion/lakeflow-connect/jira-pipeline) or [Confluence](/aws/en/ingestion/lakeflow-connect/confluence-pipeline), you may need to transform the data to match the required schema.

   note

   The Unity Catalog table must meet the following requirements:

   * The table is either a [streaming table](/aws/en/ldp/concepts/streaming-tables) or a table with [Change data feed](/aws/en/tables/features/change-data-feed) enabled.
   * The table contains a content column of `BINARY` or `STRING` type.
   * The table contains a `metadata` or `_metadata` column of `STRUCT` type with the following fields:
     + `file_path`: string
     + `file_name`: string
     + `file_size`: long
     + `file_modification_time`: timestamp

   1. Under **Type**, select **Files in a Table**.
   2. In the **Source** field, select the Unity Catalog table that contains your data.
   3. (Optional) In the **File content column** field, select the column that contains the file content. This defaults to the `content` column if not specified. Only `BINARY` and `STRING` type columns are shown.
   4. In the **Name** field, enter a name for your knowledge source.
   5. Under **Describe the content**, describe what content the knowledge source contains to help the agent understand when to use this data source.

   note

   AI Search indexes are only supported if the index uses one of the following embedding models: `databricks-gte-large-en`, `databricks-bge-large-en`, or `databricks-qwen3-embedding-0-6b`. When creating your index, ensure you select one of these embedding models. For more information, see [Create an AI Search index](/aws/en/ai-search/create-ai-search#create-vector-search-index).

   1. Under **Type**, select **AI Search Index**.
   2. In the **Source** field, select the index you want to provide the agent.
   3. In the **Name** field, enter a name for your knowledge source.
   4. In the **Doc URI Column**, select the column with a link or reference to where the information came from. The agent will use this in its citations.
   5. In the **Text Column** field, specify the column that contains the raw text you want the agent to retrieve.
   6. Under **Describe the content**, describe what content the knowledge source contains to help the agent understand when to use this data source.
4. (Optional) If you would like to add more knowledge sources, click **Add knowledge source**. You can provide up to 10 knowledge sources.
5. (Optional) In the **Instructions** field, specify guidelines for how the agent should respond.
6. Click **Create Agent**.

It can take up to a few hours to create your agent and sync the knowledge sources you provided. When the agent is ready, the right side panel displays the synced knowledge sources.

important

If you update or add files to your knowledge sources, you need to click  **Sync** for the agent to pick up the changes. Syncing is done incrementally. For example, if you add a new file to a previously synced Unity Catalog volume, syncing will only process the newly added file.

**Any user with `CAN MANAGE` permission on the knowledge assistant can sync knowledge sources.**

## Step 2: Test your agent

After your agent has finished building, test it out by chatting with it. The agent should respond with citations for questions related to its knowledge sources.

1. Start chatting with your agent directly in the **Build** tab.
2. (Optional) You can also click **Open in Playground** to chat with it in [AI Playground](/aws/en/large-language-models/ai-playground).
3. Enter a question for your agent.
4. Evaluate its response:
   1. Click **View thoughts** to see how your agent approached responding to the question.
   2. Click **View trace** to see the full trace. You can [add labels to traces](/aws/en/mlflow3/genai/human-feedback/dev-annotations) in the UI to track quality assessments during the development process.
   3. Click **View sources** to see what files the agent cites as references. This opens a side panel with a list of sources for you to review.

If you're satisfied with your agent's performance, continue using the agent as-is.

## Step 3: Improve quality

Knowledge Assistant can adjust the agent's behavior based on natural language feedback. Gather feedback from experts through the configuration page to improve your agent's quality. Collecting labeled data for your agent can improve its quality and behavior.

In the **Examples** tab, add questions your users will ask or questions your agent has previously gotten wrong. Alternatively, you can also import labeled data directly from a Unity Catalog table.

1. Add questions to label:

   1. Click **+ Add** to add a question.
   2. In the **Add a question** modal, enter your question.
   3. Click **Add**. The question should appear in the UI.
   4. Repeat until you’ve added all the questions you want to evaluate.
   5. To delete a question, click the kebab menu, then **Delete**.
2. After you've finished adding your questions, you can share the agent with others to review to help you build a high-quality labeled dataset. In the upper righthand corner, click the kebab menu  to [manage permissions](#permissions).

   In order for experts to access and provide feedback, you need to grant them the following permissions:

   * CAN\_MANAGE permission from the agent configuration page
3. Share a link to your Knowledge Assistant configuration page to gather feedback from experts.
4. To label data, click on a question and add **Guidelines** in the pane that appears. Guidelines apply right after they are saved.
5. Test the agent again in the configuration page or AI Playground to see its improved performance. If needed, add more questions and guidelines to continue improving the behavior.

### (Optional) Import and export labeled data

To import new questions and feedback directly from a Unity Catalog table:

1. Click **Import**.
2. In the **Source** field, select the Unity Catalog table containing the labeled data.

   The table must have the following schema:

   * `eval_id`: `string`
   * `request`: `string`
   * `guidelines`: `array`
     + `items`: `string`
   * `metadata`: `string`
   * `tags`: `string`
3. Click **Import**.

New questions and guidelines are merged into the labeled data table on the right.

To export labeled data as a Unity Catalog table:

1. Click **Export**.
2. In the **Schema** field, select the Unity Catalog schema location to save the data to.
3. In the **Table name** field, enter a name for the table.
4. Click Export.

A new table is created with the labeled data.

## Manage knowledge sources

After you create a knowledge assistant, you can add, remove, and update knowledge sources without recreating the agent. Only the creator of the knowledge assistant can add or remove knowledge sources. Any user with `CAN MANAGE` permission can update the knowledge source name and description, and sync knowledge sources.

You can draft multiple changes at once — for example, add a new source, remove an existing one, and update the description of another — and then apply all changes simultaneously when you click **Save and Update**.

### Add a knowledge source

1. On the agent configuration page, go to the **Sources** tab.
2. Click **+ Add** below the existing knowledge source cards.
3. Select the type of knowledge source to add (files in a Unity Catalog volume, files in a Unity Catalog table, or AI Search index) and configure it as described in [Step 1: Configure your agent](#step-1-configure-your-agent).
4. Click **Save and Update**.

The agent begins syncing the new knowledge source. Existing knowledge sources and their data are not affected.

note

You can provide up to 10 knowledge sources per knowledge assistant. Adding a file source starts an ingestion job that can take time depending on the volume of data.

### Remove a knowledge source

1. On the agent configuration page, go to the **Sources** tab.
2. Hover over the knowledge source card you want to remove and click the trash icon  in the top right corner.
3. In the confirmation dialog, click **Confirm** to remove the source.
4. Click **Save and Update**.

The knowledge source and its associated data are removed from the agent. This action is not easily reversible because re-adding the source requires re-ingesting the data.

important

You cannot remove the last remaining knowledge source. A knowledge assistant must always have at least one knowledge source.

### Update knowledge source details

You can update the name and description of an existing knowledge source. These details help the agent understand when to use each source.

1. On the agent configuration page, go to the **Sources** tab.
2. Hover over the knowledge source card you want to edit and click the pencil icon.
3. Update the **Name** or **Description** fields.
4. Click **Save and Update**.
