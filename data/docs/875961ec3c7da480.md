**Applies to:**  Databricks SQL  Databricks Runtime

Represents the untyped NULL value

## Syntax

```
{ NULL | VOID }
```

## Limits

The only value the VOID type can hold is NULL.

Delta Lake tables have the following limitations on `VOID` for write operations:

* `VOID` is not accepted inside complex types such as `ARRAY`, `MAP`, or user-defined types (UDTs).
* A table cannot have only `VOID` columns and no other types.
* A `STRUCT` cannot have only `VOID` fields and no other types.

These limitations apply only to data writing operations, not to read or metadata-only operations.

## Literals

NULL

## Examples

SQL

```
> SELECT typeof(NULL);  
  VOID  
  
> SELECT cast(NULL AS VOID);  
  VOID
```

## Delta table support

In Databricks Runtime 18.2 and above, Delta Lake supports `VOID` columns. A `VOID` column can be widened to any other data type.

`VOID` in streaming writes is not supported since the schemas must be set when using streaming.

note

In earlier versions, Delta Lake does not support the `VOID` type. See the [Databricks Runtime 18.2 release notes](/aws/en/release-notes/runtime/18.2#nulltype-void-support-in-delta-tables) for details on behavioral changes when upgrading.
