Serverless pipelines run Lakeflow Spark Declarative Pipelines on Databricks-managed compute, removing most infrastructure configuration.

Databricks recommends developing new pipelines using serverless. Some workloads might require configuring classic compute or working with the legacy Hive metastore. See [Configure classic compute for pipelines](/aws/en/ldp/configure-compute) and [Use Lakeflow Spark Declarative Pipelines with legacy Hive metastore](/aws/en/ldp/hive-metastore).

note

* Serverless pipelines always use Unity Catalog. See [Use Unity Catalog with pipelines](/aws/en/ldp/unity-catalog).
* For serverless compute limitations, see [Serverless compute limitations](/aws/en/compute/serverless/limitations). The Structured Streaming trigger limitations described there do not apply to Lakeflow Spark Declarative Pipelines pipeline modes. Serverless pipelines support triggered, continuous, and real-time modes. See [Triggered vs. continuous pipeline mode](/aws/en/ldp/pipeline-mode).
* You cannot manually add compute settings in a `clusters` object in the JSON configuration for a serverless pipeline. Attempting to do so results in an error.

* If you need to use an AWS PrivateLink connection with your serverless Lakeflow Spark Declarative Pipelines, contact your Databricks representative.

## Requirements

* Your workspace must have Unity Catalog enabled to use serverless pipelines.

* You must have accepted the serverless [terms of use](/aws/en/admin/sql/serverless#accept-terms).
* Your workspace must be in a [serverless-enabled region](/aws/en/resources/feature-region-support#serverless-aws).

## Recommended configuration for serverless pipelines

important

Cluster creation permission is not required to configure serverless pipelines. By default, all workspace users can use serverless pipelines.

Serverless pipelines remove most configuration options, as Databricks manages all infrastructure. When you create a new pipeline, the default is to use serverless. To learn how to configure a serverless pipeline, see [Configure pipelines](/aws/en/ldp/configure-pipeline).

You can also convert existing pipelines configured with Unity Catalog to use serverless. See [Convert an existing pipeline to use serverless](#convert).

## Other configuration considerations

The following configuration options are also available for serverless pipelines:

* You might choose to use the **Continuous** pipeline mode when running pipelines in production. See [Triggered vs. continuous pipeline mode](/aws/en/ldp/pipeline-mode).
* Add **Notifications** for email updates based on success or failure conditions. See [Add email notifications for pipeline events](/aws/en/ldp/monitoring-ui#notifications).
* Use the **Configuration** field to set key-value pairs for the pipeline. These configurations serve two purposes:
  + Set arbitrary parameters you can reference in your source code. See [Use parameters with pipelines](/aws/en/ldp/parameters).
  + Configure pipeline settings and Spark configurations. See [Pipeline properties reference](/aws/en/ldp/properties).
* Use the **Preview** channel to test your pipeline against pending Lakeflow Spark Declarative Pipelines runtime changes and trial new features.
* Declare external Python dependencies through the pipeline's **Environment** settings. Manually restarting the Python process (`dbutils.library.restartPython()`) is not supported, so you cannot install or reload dependencies at runtime. See [Manage Python dependencies for pipelines](/aws/en/ldp/developer/external-dependencies).

Serverless usage policies allow your organization to apply custom tags on serverless usage for granular billing attribution. After you select the **Serverless** checkbox, the **Usage policy** setting appears where you can select the policy you want to apply to the pipeline. ## Select a performance mode

For triggered pipelines, you can select the serverless compute performance mode using the **Performance optimized** setting in the pipeline scheduler. When this setting is disabled, the pipeline uses standard performance mode. Standard performance mode is designed to reduce costs for workloads where a slightly higher launch latency is acceptable. Serverless workloads using standard performance mode typically start within four to six minutes after being triggered, depending on compute availability and optimized scheduling.

When **Performance optimized** is enabled, your pipeline is optimized for performance, resulting in faster startup and execution for time-sensitive workloads.

Both modes use the same SKU, but standard performance mode consumes fewer DBUs, reflecting lower compute usage.

note

To use standard performance mode in continuous pipelines, reach out to your Databricks account team.

## Serverless pipeline features

In addition to simplifying configuration, serverless pipelines have the following features:

* **Incremental refresh for Materialized views**: Updates for materialized views are refreshed incrementally whenever possible. Incremental refresh has the same results as full recomputation. The update uses a full refresh if results cannot be computed incrementally. See [Incremental refresh for materialized views](/aws/en/ldp/incremental-refresh).

* **Stream pipelining**: To improve utilization, throughput, and latency for streaming data workloads such as data ingestion, microbatches are *pipelined*. In other words, instead of running microbatches sequentially like standard Spark Structured Streaming, serverless Lakeflow Spark Declarative Pipelines runs microbatches concurrently, improving compute resource utilization. Stream pipelining is enabled by default in serverless pipelines.
* **Vertical autoscaling**: serverless Lakeflow Spark Declarative Pipelines adds to the horizontal autoscaling provided by Databricks enhanced autoscaling by automatically allocating the most cost-efficient instance types that can run your pipeline without failing because of out-of-memory errors. See [What is vertical autoscaling?](/aws/en/ldp/auto-scaling#vertical-autoscaling)

## Convert an existing pipeline to use serverless

You can convert existing pipelines configured with Unity Catalog to serverless pipelines. Complete the following steps:

1. In your Databricks workspace's sidebar, click **Jobs & Pipelines**.
2. Click the **Name** of the pipeline.
3. Click **Settings**.
4. In the right sidebar, under **Compute**, click .
5. Check the box next to **Serverless**.
6. Click **Save**.

important

When you enable serverless, any compute settings you have configured for a pipeline are removed. If you switch a pipeline back to non-serverless updates, you must reconfigure the desired compute settings to the pipeline configuration.
