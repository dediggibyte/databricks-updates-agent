This page is a reference for Unity Catalog privileges and the securable objects they apply to.

For detailed descriptions of each securable object type, see [Unity Catalog securable objects reference](/aws/en/data-governance/unity-catalog/securable-objects). To learn how to grant privileges in Unity Catalog, see [Show, grant, and revoke privileges](/aws/en/data-governance/unity-catalog/manage-privileges/#grant).

note

This page refers to the Unity Catalog privileges and inheritance model in Privilege Model version 1.0. If you created your Unity Catalog metastore during the public preview (before August 25, 2022), you might be on an earlier privilege model that doesn't support the current inheritance model. You can upgrade to Privilege Model version 1.0 to get [privilege inheritance](/aws/en/data-governance/unity-catalog/access-control/permissions-concepts#inheritance). See [Upgrade to privilege inheritance](/aws/en/archive/unity-catalog/upgrade-privilege-model).

## Securable objects in Unity Catalog

A securable object is an object defined in the Unity Catalog [metastore](/aws/en/data-governance/unity-catalog/securable-objects#metastore) on which privileges can be granted to a principal (user, service principal, or group). Securable objects in Unity Catalog are hierarchical, from the [metastore](/aws/en/data-governance/unity-catalog/securable-objects#metastore) at the top, through [catalogs](/aws/en/data-governance/unity-catalog/securable-objects#catalog) and [schemas](/aws/en/data-governance/unity-catalog/securable-objects#schema), down to the data objects they contain ([tables](/aws/en/data-governance/unity-catalog/securable-objects#table), [views](/aws/en/data-governance/unity-catalog/securable-objects#view), [volumes](/aws/en/data-governance/unity-catalog/securable-objects#volume), [functions](/aws/en/data-governance/unity-catalog/securable-objects#function), and [models](/aws/en/data-governance/unity-catalog/securable-objects#model)). Additional securable objects govern access to external storage, external services, and OpenSharing.

For detailed descriptions of each securable object type, see [Unity Catalog securable objects reference](/aws/en/data-governance/unity-catalog/securable-objects).

## Overview of privileges in Unity Catalog

The following table summarizes the capability each Unity Catalog privilege grants. For full descriptions, see [Detailed Unity Catalog privileges reference](#privilege-descriptions).

| Privilege | Allows you to |
| --- | --- |
| [ACCESS](#access) | Use a [service credential](/aws/en/data-governance/unity-catalog/securable-objects#service-credential) to access an external service. |
| [ALL PRIVILEGES](#all-privileges) | Perform all capabilities on an object and its children. `ALL PRIVILEGES` does not include the `EXTERNAL USE SCHEMA`, `EXTERNAL USE LOCATION`, or `MANAGE` privileges. |
| [APPLY TAG](#apply-tag) | Add and edit tags on an object. For [tables](/aws/en/data-governance/unity-catalog/securable-objects#table) and [views](/aws/en/data-governance/unity-catalog/securable-objects#view), also enables column tagging. For registered [models](/aws/en/data-governance/unity-catalog/securable-objects#model), also enables version tagging. |
| [BROWSE](#browse) | Discover objects, view their metadata, and request access to them without needing `USE CATALOG` or `USE SCHEMA`. Granted at the catalog level. |
| [CREATE CATALOG](#create-catalog) | Create a [catalog](/aws/en/data-governance/unity-catalog/securable-objects#catalog) in the [metastore](/aws/en/data-governance/unity-catalog/securable-objects#metastore). |
| [CREATE CLEAN ROOM](#create-clean-room) | Create a [clean room](/aws/en/data-governance/unity-catalog/securable-objects#clean-room) for multi-party collaboration without sharing underlying data. |
| [CREATE CONNECTION](#create-connection) | Create a [connection](/aws/en/data-governance/unity-catalog/securable-objects#connection) to an external database for Lakehouse Federation. |
| [CREATE EXTERNAL LOCATION](#create-external-location) | Create an [external location](/aws/en/data-governance/unity-catalog/securable-objects#external-location) that associates a cloud storage path with a [storage credential](/aws/en/data-governance/unity-catalog/securable-objects#storage-credential). Required on both the [metastore](/aws/en/data-governance/unity-catalog/securable-objects#metastore) and the storage credential. |
| [CREATE EXTERNAL METADATA](#create-external-metadata) | Create [external metadata](/aws/en/data-governance/unity-catalog/securable-objects#external-metadata) objects for use in custom data lineage configurations. |
| [CREATE EXTERNAL TABLE](#create-external-table) | Create external tables at a specific cloud storage path using an [external location](/aws/en/data-governance/unity-catalog/securable-objects#external-location) or [storage credential](/aws/en/data-governance/unity-catalog/securable-objects#storage-credential). |
| [CREATE EXTERNAL VOLUME](#create-external-volume) | Create external [volumes](/aws/en/data-governance/unity-catalog/securable-objects#volume) using an [external location](/aws/en/data-governance/unity-catalog/securable-objects#external-location). |
| [CREATE FEATURE](#create-feature) | Create a [feature](/aws/en/data-governance/unity-catalog/securable-objects#feature) in a [schema](/aws/en/data-governance/unity-catalog/securable-objects#schema). |
| [CREATE FOREIGN CATALOG](#foreign-catalog) | Create foreign [catalogs](/aws/en/data-governance/unity-catalog/securable-objects#catalog) using a Lakehouse Federation [connection](/aws/en/data-governance/unity-catalog/securable-objects#connection). |
| [CREATE FOREIGN SECURABLE](#create-foreign-securable) | Specify authorized cloud storage paths when creating a foreign [catalog](/aws/en/data-governance/unity-catalog/securable-objects#catalog). |
| [CREATE FUNCTION](#function) | Create [functions](/aws/en/data-governance/unity-catalog/securable-objects#function) in a [schema](/aws/en/data-governance/unity-catalog/securable-objects#schema). |
| [CREATE MANAGED STORAGE](#create-managed-storage) | Set a custom storage location for managed tables at the [catalog](/aws/en/data-governance/unity-catalog/securable-objects#catalog) or [schema](/aws/en/data-governance/unity-catalog/securable-objects#schema) level, overriding the [metastore](/aws/en/data-governance/unity-catalog/securable-objects#metastore) default. |
| [CREATE MATERIALIZED VIEW](#create-materialized-view) | Create [materialized views](/aws/en/data-governance/unity-catalog/securable-objects#materialized-view) in a [schema](/aws/en/data-governance/unity-catalog/securable-objects#schema). |
