This page details how you can discover and explore catalogs, schemas, tables, and other database objects in Unity Catalog. The instructions in this page focus on returning details for database objects that you have at least the `BROWSE` or `SELECT` privilege on.

For general information on Unity Catalog privileges, see [Unity Catalog privileges reference](/aws/en/data-governance/unity-catalog/access-control/privileges-reference). For information about how to set schema ownership and permissions, see [Manage object ownership](/aws/en/data-governance/unity-catalog/manage-privileges/#manage-ownership) and [Manage privileges in Unity Catalog](/aws/en/data-governance/unity-catalog/manage-privileges/).

This page includes instructions for Catalog Explorer and SQL. Select the  **Catalog** icon in the workspace side bar to access Catalog Explorer. You can execute SQL commands from a notebook or the query editor attached to compute. To view database objects with Catalog Explorer, you must have at least the `BROWSE` privilege on the objects. To view database objects with SQL, you must have at least the `SELECT` privilege on the object, as well as `USE CATALOG` on the parent catalog and `USE SCHEMA` on the parent schema. You can navigate Unity Catalog-governed database objects in Catalog Explorer without active compute. To explore data in the `hive_metastore` and other catalogs not governed by Unity Catalog, you must attach to compute with appropriate privileges.

## Filtering database objects

During interactive exploration of database objects with Catalog Explorer, you can use the provided text box to filter results. Matched strings in object names are highlighted, but only among currently visible database objects. For complete search of all database objects, see [Search for workspace objects](/aws/en/search/).

SQL provides similar functionality by optionally specifying a `regex_pattern` clause in conjunction with a `SHOW` statement, such as the following:

SQL

```
SHOW TABLES IN schema_name LIKE 'sales_*_fy23'
```

## Request privileges on an object

Preview

This feature is in [Public Preview](/aws/en/release-notes/release-types).

If access request destinations are configured in Unity Catalog, you can request privileges on objects where you have either the `BROWSE` privilege or a direct URL to the object. You can also request privileges directly from permission denied errors in notebooks and the SQL editor.

If you have `ALL PRIVILEGES` on an object, you cannot request access on behalf of others using Catalog Explorer. Instead, use the [Create Access Requests API](https://docs.databricks.com/api/workspace/rfa/batchcreateaccessrequests).

1. In your Databricks workspace, click  **Catalog**.
2. Select the object, such as a catalog, schema, table, or view.
3. Click **Request permissions**.
4. Select the principals you are requesting access for. You can request access for yourself, service principals, groups, and other users.
5. Select the privileges and optionally add a message to approvers.

   If a redirect URL is configured, you won’t see this form. Instead, you are redirected to the specified URL to complete your request.
6. Click **Request**. A message is sent to the configured approvers with your access request.

## Explore catalogs

Catalogs represent the top level of data governance in each Unity Catalog metastore.

* SQL
* Catalog Explorer

Run the following command to see a list of catalogs available to you.

SQL

```
SHOW CATALOGS
```

See [SHOW CATALOGS](/aws/en/sql/language-manual/sql-ref-syntax-aux-show-catalogs).

When you access Catalog Explorer, you see a list of catalogs available to you.

### Select a catalog

* SQL
* Catalog Explorer

Run the following command to set your currently active catalog.

SQL

```
USE CATALOG catalog_name
```

See [USE CATALOG](/aws/en/sql/language-manual/sql-ref-syntax-ddl-use-catalog).

Click on a catalog name to select it.

### See catalog details

* SQL
* Catalog Explorer

Run the following command to describe a catalog.

SQL

```
DESCRIBE CATALOG catalog_name
```

See [DESCRIBE CATALOG](/aws/en/sql/language-manual/sql-ref-syntax-aux-describe-catalog).

Select the **Details** tab to review catalog details.

## Explore schemas

Schemas are collections of tables, views, volumes, functions, and models in Unity Catalog. Schemas are contained in catalogs.

* SQL
* Catalog Explorer

Run the following command to see a list of schemas available to you.

SQL

```
SHOW SCHEMAS IN catalog_name
```

See [SHOW SCHEMAS](/aws/en/sql/language-manual/sql-ref-syntax-aux-show-schemas).

When you select a catalog in Catalog Explorer, you see a list of available schemas.

### Select a schema

* SQL
* Catalog Explorer

Run the following command to set your currently active schema.

SQL

```
USE schema catalog_name.schema_name
```

See [USE SCHEMA](/aws/en/sql/language-manual/sql-ref-syntax-ddl-use-schema).

Click on a schema name to select it.

### See schema details

* SQL
* Catalog Explorer

Run the following command to describe a schema.

SQL

```
DESCRIBE SCHEMA schema_name
```

See [DESCRIBE SCHEMA](/aws/en/sql/language-manual/sql-ref-syntax-aux-describe-schema).

Select the **Details** tab to review schema details.

## Explore tables and views

Tables and views are contained in schemas. Most tables in Unity Catalog are backed by Delta Lake, but you might also have access to tables registered against external data. See [What data can you query with Databricks?](/aws/en/query/#lakehouse-external).

Views in Unity Catalog always reference data in another table.

* SQL
* Catalog Explorer

Run the following command to see a list of tables available to you.

SQL

```
SHOW TABLES IN catalog_name.schema_name
```

Run the following command to see a list of tables available to you.

SQL

```
SHOW VIEWS IN catalog_name.schema_name
```

See [SHOW TABLES](/aws/en/sql/language-manual/sql-ref-syntax-aux-show-tables) and [SHOW VIEWS](/aws/en/sql/language-manual/sql-ref-syntax-aux-show-views).

When you select a schema in Catalog Explorer, you see a list of available tables and views.

note

If the schema has other database objects like volumes present, you might need to click **Tables** to expand the list of tables and views.

## View table contents and details

You can view most table details with either Catalog Explorer or SQL. Some details are only available in the Catalog Explorer UI.

Select a table in Catalog Explorer to explore table details.

### Explore table columns

note

Users with only the `BROWSE` privilege on an object cannot explore table columns using the `SHOW COLUMNS` SQL command.

* SQL
* Catalog Explorer

Run the following command to view table columns.

SQL

```
SHOW COLUMNS IN table_name
```

See [SHOW COLUMNS](/aws/en/sql/language-manual/sql-ref-syntax-aux-show-columns).

Select the **Columns** tab to view table columns.

### View sample data

* SQL
* Catalog Explorer

Run the following command to view 1000 records from a table.

SQL

```
SELECT * FROM table_name LIMIT 1000;
```

See [Query data](/aws/en/query/).

Select the **Sample Data** tab to view sample data. You must have access to active compute to sample data.

### Explore table data using an LLM

You can ask natural language questions about your data using Sample Data Explorer in Catalog Explorer.

For any dataset, click the **Sample Data** tab and ask a question about the data. Genie generates SQL using table metadata and row-level sample data as context. After the query is generated, you can validate and run it against the underlying table. You can also enable auto-run to execute generated SQL automatically each time. You must have access to an active compute resource and [Partner-powered AI features](/aws/en/databricks-ai/partner-powered) enabled.

For more information, see [Explore sample data with Genie Code](/aws/en/genie-code/sample-data-explorer).

### See table details

note

Users with only the `BROWSE` privilege on an object cannot view table details using the `DESCRIBE TABLE` SQL command.

* SQL
* Catalog Explorer

Run the following command to describe a table.

SQL

```
DESCRIBE TABLE table_name
```

Run the following command to display table properties for a table.

SQL

```
SHOW TBLPROPERTIES table_name
```

See [DESCRIBE TABLE](/aws/en/sql/language-manual/sql-ref-syntax-aux-describe-table) and [SHOW TBLPROPERTIES](/aws/en/sql/language-manual/sql-ref-syntax-aux-show-tblproperties).

Select the **Details** tab to review table details, including table properties.

### View table history

Table history is available for Delta tables. All Unity Catalog managed tables are Delta tables.

* SQL
* Catalog Explorer

Run the following command to review table history.

SQL

```
DESCRIBE HISTORY table_name
```

See [DESCRIBE HISTORY](/aws/en/sql/language-manual/delta-describe-history).

Select the **History** tab to review table history.

### View frequent queries and users

If the table is registered in Unity Catalog, you can view the most frequent queries made on the table and users who accessed the table in the past 30 days using Catalog Explorer. See [View table insights and popularity](/aws/en/discover/table-insights).

## View primary key and foreign key relationships

For tables with foreign keys defined, click **View relationships**  at the top-right of the **Columns** tab. The Entity Relationship Diagram (ERD) opens. The ERD displays the primary key and foreign key relationships between tables in a graph, providing a clear and intuitive representation of how data entities connect.

For more information about primary key and foreign key constraints, see [Constraints on Databricks](/aws/en/tables/constraints). For more information about entity relationship diagrams, see [View the Entity Relationship Diagram](/aws/en/catalog-explorer/entity-relationship-diagram).
