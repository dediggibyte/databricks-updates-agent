Genie Code helps you write, debug, and understand code in your Databricks notebooks using natural-language prompts, slash commands, and AI-based autocomplete.

## What can Genie Code help with?

Genie Code is an AI assistant you interact with conversationally in Databricks. Describe a task in English, and Genie Code generates Python code or SQL queries, explains complex code, and fixes errors automatically. Genie Code uses Unity Catalog metadata to understand your tables, columns, descriptions, and popular data assets across your company and provide personalized responses.

Genie Code can help you with the following tasks:

* Generate code.
* Debug code, including identifying and suggesting fixes for errors.
* Transform, optimize, and refactor code.
* Explain code.
* Help you find relevant information in the Databricks documentation.

For information about the models that power Genie Code, see [Partner-powered AI features](/aws/en/databricks-ai/partner-powered). For general information about Genie Code, see [Genie Code](/aws/en/genie-code/).

## Run code in Genie Code on serverless compute

You can run code directly from the Genie Code side panel. Genie Code uses serverless compute by default, which is the default compute on Databricks.

If you’re on a page that already has a compute resource selected (for example, notebooks or the SQL editor), Genie Code automatically uses that compute instead. For information about compute types, see [Compute](/aws/en/compute/).

If you don't have access to serverless compute, you must have a compute instance available to run code in the Genie Code panel.

## Genie Code command shortcuts for notebooks

In a notebook, Genie Code is available in the Genie Code pane or inline in a code cell.

To use Genie Code directly in a code cell, press `Cmd+I` on MacOS or `Ctrl+I` on Windows. A text box appears in the cell. You can enter a question or comment in English and then press `Enter` (not `Shift+Enter`, like you would to run a cell) to have Genie Code generate a response.

### Use slash commands for prompts

Slash commands are a shortcut for creating common prompts.

| Prompt text | What Genie Code does |
| --- | --- |
| `/` | Displays common commands |
| `/doc` | Comments the code in a diff view |
| `/explain` | Provides an explanation of the code in a cell |
| `/fix` | Proposes a fix to any code errors in a diff view |
| `/findTables` | Searches for relevant tables based on Unity Catalog metadata. To find features or feature tables, mention “features” or “feature tables” in the query. See [Find features and feature tables using Genie Code](/aws/en/machine-learning/feature-store/uc/ui-uc#features). |
| `/findQueries` | Searches for relevant queries based on Unity Catalog metadata. |
| `/optimize` | Suggests performance improvements for SQL queries. |
| `/prettify` | Formats SQL queries for readability. |

When you use `/fix` or `/doc`, in the diff window select **Accept** to accept the proposed changes or **Reject** to keep the original code. If you accept the proposed code, the code does not automatically run. You can review the code before running it. If the generated code is not what you wanted, try again by adding more details or information to your comment. See [Tips to improve Genie Code responses](/aws/en/genie-code/tips).

Get detailed explanations of code snippets. Use the `/explain` prompt and include terms like “be concise” or “explain code line-by-line” to request the level of detail that you want. You can also ask Genie Code to add comments to the code.

For code autocomplete, performance may be better using the Genie Code pane than in a notebook cell.

Genie Code closes automatically if you **Accept** or **Reject** the code it generated.

### Reference tables in prompts using `@`

To quickly reference tables in Genie Code prompts, use the `@` (at) symbol.

### Debug environment errors

Genie Code can help diagnose and suggest fixes for environment failures, including library installation errors.

When there is an installation failure in the environment side panel, the Genie Code icon  appears in the bottom right. Click  to open the Genie Code pane, where Genie Code runs `/repairEnvironment` to help diagnose and fix the issue.

You can also open the pip logs and then click **Diagnose error** to run the `/repairEnvironment` command in the Genie Code pane.

### Optimize Python, PySpark, and SQL code

Using the slash prompt `/optimize`, evaluate and optimize Python, PySpark, and SQL code.

1. In a notebook cell or the SQL editor, click the  Genie Code icon.
2. In the prompt box, type `/optimize`, and click the **Generate** button.
3. To use the optimization suggestions from Genie Code, click **Accept**.

Here's how `/optimize` helps with code:

* **Pre-run warnings:** Yellow underlines highlight opportunities to optimize your query before execution, such as when you are not filtering on a partition column. Hover over the underlined text to view optimization suggestions.
* **Post-run analysis:** After you run the code, click the **Optimize** button.
* **Table-level suggestions:** For example, when users create a table, they receive a hint recommending `CREATE TABLE CLUSTER BY AUTO`.
