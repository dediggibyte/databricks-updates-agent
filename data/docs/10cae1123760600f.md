Databricks workspaces can be hosted on Amazon AWS, Microsoft Azure, and Google Cloud Platform. You can use Databricks on any of these hosting platforms to access data wherever you keep it, regardless of cloud.

This page lists the regions supported by Databricks on AWS.

note

Some Databricks features, known as [Designated Services](/aws/en/resources/designated-services), use [Databricks Geos](/aws/en/resources/databricks-geos) to manage data residency when processing customer content. A Geo is a group of regions that Databricks uses to give customers predictability and transparency about where data is processed.

For more region-related information, see the following pages:

* [Features with limited regional availability](/aws/en/resources/feature-region-support).
* [IP addresses and domains for Databricks services and assets](/aws/en/resources/ip-domain-region).
