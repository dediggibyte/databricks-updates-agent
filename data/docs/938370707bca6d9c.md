This page describes how to use the Unity REST API to create, read, and write to Unity Catalog managed and external tables from external Delta clients. For a full list of supported integrations, see [Unity Catalog integrations](/aws/en/external-access/integrations).

## Create, read, and write using the Unity REST API

Preview

Creating and writing to Unity Catalog managed tables from Delta clients is in [Public Preview](/aws/en/release-notes/release-types).

The Unity REST API provides external clients create, read, and write access to tables registered to Unity Catalog. Configure access using the [workspace URL](https://docs.databricks.com/aws/en/workspace/workspace-details#workspace-url) as the endpoint. The following table types are accessible:

| Table type | Read | Write | Create |
| --- | --- | --- | --- |
| Managed Delta | Yes | Yes\* | Yes\* |
| External Delta | Yes | Yes | Yes |

\* Supported for managed Delta tables with [catalog commits](/aws/en/tables/features/catalog-commits).

warning

To avoid potential data corruption and data loss issues, Databricks recommends you do not modify the same Delta Lake table stored in S3 from different writer clients.

## Supported clients

Apache Spark is supported for creating, reading, and writing to Unity Catalog managed and external Delta tables using the Unity REST API.

The following clients are supported in Beta. Contact your Databricks account team if you are interested in using them:

* Apache Flink
* DuckDB
* StreamNative
* Starburst

## Requirements

Databricks supports Unity REST API access to tables as part of Unity Catalog. You must have Unity Catalog enabled in your workspace to use these endpoints.

You must also complete the following configuration steps to configure access to tables from Delta clients using the Unity REST API:

* Enable *External data access* for your metastore. See [Enable external data access on the metastore](/aws/en/external-access/admin#external-data-access).
* Grant the principal accessing data externally the `EXTERNAL USE SCHEMA` privilege on the schema containing the objects. See [Grant a principal Unity Catalog privileges](/aws/en/external-access/admin#external-schema).
* For external tables accessed by path: Grant the principal the `EXTERNAL USE LOCATION` privilege on the external location containing the table path. See [Grant a principal Unity Catalog privileges](/aws/en/external-access/admin#external-schema).
* Make sure that the principal has relevant privileges:
  + `SELECT` on the table for reads
  + `MODIFY` on the table for writes
  + `CREATE` on the schema for table creation
  + For external writes to managed Delta tables, verify that the table being written to has [catalog commits](/aws/en/tables/features/catalog-commits) enabled.
* Authenticate using one of the following methods:
  + Personal access token (PAT): See [Authorize access to Databricks resources](/aws/en/dev-tools/auth/).
  + OAuth machine-to-machine (M2M) authentication: Supports automatic credential and token refresh for long-running Spark jobs (>1 hour). See [Authorize service principal access to Databricks with OAuth](/aws/en/dev-tools/auth/oauth-m2m).

## Limitations

* External access to [UniForm](https://docs.databricks.com/aws/en/delta/uniform) tables with IcebergCompatV3 is not currently supported.
* External clients cannot alter `delta.*` or `databricks.*` table properties, add or remove table features, rename columns, and add or drop `CHECK` constraints.
* External clients can't perform table maintenance operations, such as [`OPTIMIZE`](https://docs.databricks.com/aws/en/sql/language-manual/delta-optimize), [`VACUUM`](/aws/en/sql/language-manual/delta-vacuum), and [`ANALYZE`](https://docs.databricks.com/aws/en/sql/language-manual/sql-ref-syntax-aux-analyze-compute-statistics), on managed Delta tables.
* External clients cannot create tables with generated columns, default columns, or constraint columns.
* When creating external tables, Databricks recommends using Apache Spark to ensure that column definitions are in a format compatible with Apache Spark. The API does not validate the correctness of the column specification. If the specification is not compatible with Apache Spark, then Databricks Runtime might be unable to read the tables.

note

To read tables with [row filters or column masks](/aws/en/data-governance/unity-catalog/filters-and-masks/) attached from an external Delta client, see [Cross-engine attribute-based access controls (ABAC)](/aws/en/external-access/cross-engine-abac) for required client versions and configuration.

## Access Delta tables with Apache Spark using PAT authentication

PAT authentication for external Spark clients requires:

* Unity Catalog Spark client version 0.5.0 or above (`io.unitycatalog:unitycatalog-spark`)
* Apache Spark 4.0 or 4.1
* Delta Spark 4.3.0 or above
* A personal access token for the principal accessing Unity Catalog. See [Authorize access to Databricks resources](/aws/en/dev-tools/auth/).

The following configuration is required to read or write to Unity Catalog managed and external Delta tables with Apache Spark using PAT authentication:

```
"spark.sql.extensions": "io.delta.sql.DeltaSparkSessionExtension",  
"spark.sql.catalog.spark_catalog": "io.unitycatalog.spark.UCSingleCatalog",  
"spark.sql.catalog.<uc-catalog-name>": "io.unitycatalog.spark.UCSingleCatalog",  
"spark.sql.catalog.<uc-catalog-name>.uri": "<workspace-url>",  
"spark.sql.catalog.<uc-catalog-name>.token": "<token>",  
"spark.sql.defaultCatalog": "<uc-catalog-name>",  
"spark.hadoop.fs.s3.impl": "org.apache.hadoop.fs.s3a.S3AFileSystem",  
"spark.jars.packages": "io.delta:delta-spark_4.1_2.13:4.3.0,io.unitycatalog:unitycatalog-spark_4.1_2.13:0.5.0,org.apache.hadoop:hadoop-aws:3.4.2"
```

Substitute the following variables:

* `<uc-catalog-name>`: The name of the catalog in Unity Catalog that contains your tables.
* `<token>`: Personal access token (PAT) for the principal configuring the integration.

* `<workspace-url>`: The Databricks [workspace URL](/aws/en/workspace/workspace-details#workspace-url). For example, `cust-success.cloud.databricks.com`.

note

The package versions shown above are current as of the last update to this page. Newer versions might be available. Verify that package versions are compatible with your Spark version.

For additional details about configuring Apache Spark for cloud object storage, see the [Unity Catalog OSS documentation](https://docs.unitycatalog.io/integrations/unity-catalog-spark/#optional-running-spark-sql-for-cloud-object-stores).

important

Databricks Runtime 16.4 and above is required to read from, write to, or create tables with catalog commits enabled.
Databricks Runtime 18.0 and above is required to enable or disable catalog commits on existing tables.

To create managed Delta tables with [catalog commits](/aws/en/tables/features/catalog-commits), use the following SQL:

SQL

```
CREATE TABLE <uc-catalog-name>.<schema-name>.<table-name> (id INT, desc STRING)  
TBLPROPERTIES ('delta.feature.catalogManaged' = 'supported') USING delta;
```

To create external Delta tables, use the following SQL:

SQL

```
CREATE TABLE <uc-catalog-name>.<schema-name>.<table-name> (id INT, desc STRING)  
USING delta  
LOCATION <path>;
```

## Access Delta tables with Apache Spark using OAuth authentication

Databricks also supports OAuth machine-to-machine (M2M) authentication. OAuth automatically handles token and credential renewal for Unity Catalog authentication.

OAuth authentication for external Spark clients requires:

* Unity Catalog Spark client version 0.5.0 or later (`io.unitycatalog:unitycatalog-spark`)
* Apache Spark 4.0 or 4.1
* Delta Spark 4.3.0 or later
* An OAuth M2M service principal with appropriate permissions. See [Authorize service principal access to Databricks with OAuth](/aws/en/dev-tools/auth/oauth-m2m).

The following configuration is required to create, read, or write to Unity Catalog managed tables and external Delta tables with Apache Spark using OAuth authentication:

```
"spark.sql.extensions": "io.delta.sql.DeltaSparkSessionExtension",  
"spark.sql.catalog.spark_catalog": "io.unitycatalog.spark.UCSingleCatalog",  
"spark.sql.catalog.<uc-catalog-name>": "io.unitycatalog.spark.UCSingleCatalog",  
"spark.sql.catalog.<uc-catalog-name>.uri": "<workspace-url>",  
"spark.sql.catalog.<uc-catalog-name>.auth.type": "oauth",  
"spark.sql.catalog.<uc-catalog-name>.auth.oauth.uri": "<oauth-token-endpoint>",  
"spark.sql.catalog.<uc-catalog-name>.auth.oauth.clientId": "<oauth-client-id>",  
"spark.sql.catalog.<uc-catalog-name>.auth.oauth.clientSecret": "<oauth-client-secret>",  
"spark.sql.defaultCatalog": "<uc-catalog-name>",  
"spark.hadoop.fs.s3.impl": "org.apache.hadoop.fs.s3a.S3AFileSystem",  
"spark.jars.packages": "io.delta:delta-spark_4.1_2.13:4.3.0,io.unitycatalog:unitycatalog-spark_4.1_2.13:0.5.0,org.apache.hadoop:hadoop-aws:3.4.2"
```

Substitute the following variables:

* `<uc-catalog-name>`: The name of the catalog in Unity Catalog that contains your tables.
* `<oauth-token-endpoint>`: OAuth token endpoint URL. To construct this URL:
  1. Locate your Databricks [workspace URL](/aws/en/workspace/workspace-details#workspace-url).
  2. Use the format: `https://<workspace-url>/oidc/v1/token`
* `<oauth-client-id>`: OAuth client ID for your service principal. See [Authorize service principal access to Databricks with OAuth](/aws/en/dev-tools/auth/oauth-m2m).
* `<oauth-client-secret>`: OAuth client secret for your service principal. See [Authorize service principal access to Databricks with OAuth](/aws/en/dev-tools/auth/oauth-m2m).

* `<workspace-url>`: The Databricks [workspace URL](/aws/en/workspace/workspace-details#workspace-url). For example, `cust-success.cloud.databricks.com`.

note

The package versions shown above are current as of the last update to this page. Newer versions might be available. Verify that package versions are compatible with your Spark version.
