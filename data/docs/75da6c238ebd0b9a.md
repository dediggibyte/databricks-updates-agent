[Lovable](https://lovable.dev/) is a no-code application platform that lets teams build and deploy apps without writing code. With the Databricks integration, Lovable-hosted apps can query data stored in your Databricks lakehouse using the Databricks REST API and OAuth machine-to-machine (M2M) authentication.

For example, you can build forecasting dashboards or reporting apps backed by lakehouse data.

## Prerequisites

Before you configure the integration, you need the following:

* A Databricks workspace. See [Create a workspace](/aws/en/admin/workspace/).
* A Databricks [service principal](/aws/en/admin/users-groups/manage-service-principals) with access to the workspace.
* An [OAuth secret](/aws/en/dev-tools/auth/oauth-m2m#create-secret) for the service principal. Save the **client ID** and **secret**. The secret is shown only once.
* An existing [SQL warehouse](/aws/en/compute/sql-warehouse/create) in your workspace. The service principal must have access to the SQL warehouse.
* A [Lovable](https://lovable.dev/) account.

## Step 1: Get the Databricks connection details

To connect from Lovable, you need the following values from your Databricks workspace:

* **Workspace URL**: The URL of your Databricks workspace, such as `https://dbc-1234567890123456.cloud.databricks.com`. See [Get identifiers for workspace objects](/aws/en/workspace/workspace-details).
* **Client ID**: The application ID of your service principal. This is the same value as the client ID displayed when you generated the OAuth secret.
* **Client secret**: The OAuth secret you generated for the service principal.

## Step 2: Configure the connection in Lovable

To connect your Lovable app to Databricks, see [Connect your app to Databricks](https://docs.lovable.dev/integrations/databricks) in the Lovable documentation.

Lovable stores these credentials as encrypted environment variables and passes them to your app at runtime.

## Step 3: Query Databricks data

After you configure the connection, create a Lovable app that queries Databricks data. When you describe the app you want to build, the Lovable agent automatically discovers available SQL warehouses and uses the Databricks REST API to run SQL queries.
